## Quick start

```python
from tikzpics import TikzFigure

fig = TikzFigure()
a = fig.add_node(0,  0, content="A", shape="circle", fill="cyan!40", minimum_size="1cm")
b = fig.add_node(3,  0, content="B", shape="circle", fill="cyan!40", minimum_size="1cm")
c = fig.add_node(0, -2, content="C", shape="circle", fill="cyan!40", minimum_size="1cm")
d = fig.add_node(3, -2, content="D", shape="circle", fill="cyan!40", minimum_size="1cm")

fig.draw([a, b], arrows="->")
fig.draw([a, c], arrows="->")
fig.draw([c, d], arrows="->")
fig.draw([a, d], arrows="->", color="gray", options=["dashed"])

fig.show()
```

  <TabItem label="API Reference">
    The main entry point is `TikzFigure`:

    ```python
    from tikzpics import TikzFigure

    fig = TikzFigure()

    # Add nodes
    n = fig.add_node(x, y, content="...", shape="circle", fill="blue!40")

    # Connect nodes
    fig.draw([n1, n2], arrows="->", color="gray")

    # Export
    fig.savefig("output.pdf")   # or .png, .tikz
    fig.show()                  # display inline (Jupyter)
    ```

## Features

<CardGrid stagger>
  <Card title="Python API" icon="pencil">
    Build TikZ figures via a clean Python API — no LaTeX knowledge required.
  </Card>
  <Card title="Multiple export formats" icon="document">
    Export to LaTeX source, PDF, PNG, or JPG.
  </Card>
  <Card title="Layer support" icon="list-format">
    Z-ordering via named layers using `pgfonlayer`.
  </Card>
  <Card title="3D figures" icon="star">
    3D figure support via `pgfplots`.
  </Card>
  <Card title="Jupyter magic" icon="laptop">
    Inline rendering in Jupyter/IPython with `%%tikz`.
  </Card>
  <Card title="Readable output" icon="open-book">
    Inspectable, human-readable LaTeX output.
  </Card>
</CardGrid>
